# Visual Directions v0

Objetivo: producir imagenes que hagan entendible la escala y el hilo conductor antes de construir una web real.

## Direccion A - Timeline / constelacion

Una lamina editorial con eje enero-julio y ramificaciones:

- estudio -> herramientas situadas;
- escritura -> BookOps;
- BookOps -> Ontahi;
- planes -> Atlas.

Uso ideal: primera imagen de LinkedIn y hero del sitio.

Sensacion buscada: "esto no fue una lista, fue una evolucion".

## Direccion B - Lab log con evidencia

Una pagina larga tipo cuaderno tecnico:

- fechas verificadas por git;
- commits y volumen;
- capturas con callouts;
- links a demos/repos;
- notas marginales sobre decisiones.

Uso ideal: pagina principal del sitio.

Sensacion buscada: "puedo auditarlo; no es humo".

## Direccion C - Caso de proyecto

Pagina repetible para cada artefacto:

- problema vivido;
- herramienta construida;
- stack explorado;
- rol de IA;
- evidencia;
- demo y media.

Uso ideal: deep dives por proyecto.

Sensacion buscada: "hay producto, hay arquitectura, hay oficio".

## Direccion D - Carousel LinkedIn

4 a 5 laminas:

1. seis meses construyendo herramientas para pensar con IA;
2. herramientas de estudio reales;
3. BookOps como infraestructura;
4. Ontahi/Atlas como salto conceptual;
5. modo de produccion: IA + arquitectura + revision + sistemas vivos.

Sensacion buscada: "a la mierda, quiero abrir los links".

## Copy eje

No usar: "vibe coding" como unico marco.

Usar:

> No escribi cada linea a mano. Diseñe sistemas, dirigi iteraciones, revise arquitectura y use IA como material de construccion.

## Paleta inicial

- Fondo: papel blanco roto muy leve, casi blanco.
- Tinta: grafito/negro.
- Acentos: rojo ceibo, verde hoja, azul/cobalto discreto.
- Lineas: finas, tecnicas, con labels chicos.

Evitar estetica SaaS generica, hero marketing, tarjetas enormes sin evidencia y gradientes decorativos.
